Top gap rules

Five rules every Rust engineer here would benefit from

1

Use select! to wait for signal or CLI completion

Race signal handlers against the work future so cleanup (e.g. dropping a tempdir) happens on either exit path.

control-flowPR #3600
2

Don't panic in library code

Handle all error cases explicitly — surface them as Result variants. Panics in library code remove the caller's choice.

panic-disciplinePR #3909
3

Use Option<T> for absence, not enum tags

Distinguish absence from presence with Option<T>; don't bake "missing" into the variant set of a domain enum.

type-safetyPR #4172
4

Preserve documentation when refactoring

If you remove a doc comment, add a replacement. Refactors shouldn't silently strip the rationale future readers depend on.

semantic-correctnessPR #3785
5

Use precise domain terminology in user-facing strings

e.g. say "targets" rather than "artifacts" when paths aren't available — the word the user sees should match what the system actually knows.

user-facing-messagesPR #4232

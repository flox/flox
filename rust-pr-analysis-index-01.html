Flox Rust PR Analysis — Artifact Index

Landing page for everything produced by the rust-pr-analysis-skill worktree · merged-PR window 2025-09-18 → 2026-05-15

This project mines 6-8 months of merged Rust PRs in flox/flox to extract review-validated coding rules. The analysis window is 2025-09-18 → 2026-05-15, covering 216 PRs, 944 classified comments, and 488 findings. The pipeline runs without a paid API key by orchestrating Haiku subagents in batches from inside the Claude Code session itself. Synthesis (Sonnet 4.6) produced two reusable Claude Code skills and four per-tree CLAUDE.md files.

Underlying machinery

  • scripts/pr-analysis/ — Pipeline code (~10 ingest/classify/aggregate scripts + lib/ modules).
  • scripts/pr-analysis/data/pr_analysis.db — SQLite snapshot powering all the HTML reports.
  • scripts/pr-analysis/lib/md2html.py — Pure-stdlib markdown-to-HTML helper used to render every markdown artifact in this index.
  • scripts/pr-analysis/build_dashboard.py — Regenerates rust-pr-analysis-dashboard-01.html.
  • scripts/pr-analysis/build_noise_deep_dive.py — Regenerates rust-pr-analysis-noise-deep-dive-01.html.
  • scripts/pr-analysis/build_results_html.py — Regenerates findings/results-summary.html.
  • scripts/pr-analysis/build_journey_html.py — Regenerates findings/journey-report.html.
  • scripts/pr-analysis/build_index_and_pipeline.py — Regenerates this index, the pipeline page, and every markdown-to-HTML rendering in the manifest.

Re-running aggregate_findings.py may shift cluster boundaries because greedy single-pass embedding clustering is order-dependent; pin comment_id ordering if you need reproducibility.

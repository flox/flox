Key lessons surfaced during the session

  1. Plan bugs are normal, especially around external API schemas. gh pr list --json doesn't expose mergeCommitOid or author.type — both were assumed by the original plan. Implementer subagents that adapt cleanly and flag deviations are more valuable than ones that blindly follow.
  2. Heuristic calibration is a swing problem. AGENTS.md matching went 100% (useless-true) → 0% (useless-false) → 73% (defensible) across three iterations of the same idea, each expressed differently (substring-on-section-names → Jaccard-on-full-text → key-token-on-per-section).
  3. Embeddings beat Jaccard for one-line rules. Even paraphrased duplicates Jaccard at ~0.3–0.4; MiniLM cosine puts them at ~0.55–0.65. Threshold choice is empirical.
  4. INSERT OR REPLACE with ON DELETE CASCADE is a footgun. SQLite implements REPLACE as DELETE+INSERT, which cascades to children. Use INSERT … ON CONFLICT(id) DO UPDATE SET ….
  5. Subagent self-doubt is real. A Haiku subagent failed on a 70 KB Read claiming token-limit issues while three sibling agents handled 58–74 KB files fine. Smaller batches (15 items vs 30) eliminated the trigger entirely.
  6. Calibrating on one window risks overfitting. The recent 15-PR window was 52% activations from one fix series. The iter-4 wider-window run is the generalization check.
